## 二、问题与解决方案复盘

### 2.1 技术难点

#### 问题1：后端 Supabase 配置缺失导致 502 错误

**现象**：
```
502 Bad Gateway
```

**排查过程**：
1. 检查后端日志发现 Supabase 配置为 undefined
2. 发现 `.env` 文件未正确配置
3. 确认 `config/index.js` 未读取环境变量

**解决方案**：
- 创建 `backend/.env` 文件并配置 Supabase URL 和 Key
- 重启后端服务

---

#### 问题2：登录返回 500 错误（用户已注册）

**现象**：
```
500 Internal Server Error - User already registered
```

**分析**：
- 这是正常业务逻辑，用户使用已注册邮箱登录
- 前端未正确处理错误提示

**解决方案**：
- 后端返回友好错误信息
- 前端添加错误提示优化

---

#### 问题3：体质提交返回 401 未授权

**现象**：
```
401 Unauthorized
```

**排查过程**：
1. 检查请求头发现 token 名称不匹配
2. 前端使用 `localStorage.getItem('token')`
3. 后端中间件检查 `localStorage.getItem('xiaogancao_token')`

**解决方案**：
- 修改 `frontend/src/api/request.js` 使用正确的 key 名称

---

#### 问题4：养生方案 API 数据格式不匹配

**现象**：
前端期望 `dietAdvice`，后端返回 `diet_advice`

**解决方案**：
- 统一后端返回 snake_case 格式
- 前端使用 `toCamelCase` 转换或直接适配

---

#### 问题5：数字人组件不显示

**现象**：
用户反馈 AvatarSpeaker 组件不可见

**排查过程**：
1. 检查 CSS 变量可能未加载
2. 使用固定颜色值替换 CSS 变量
3. 验证组件是否正确引入

**解决方案**：
- 使用固定颜色值替代 CSS 变量确保渲染

---

### 2.2 架构优化

#### 问题6：古风数字人形象不够美观

**原方案**：Three.js 程序化几何体

**问题**：过于简陋，不符合"温婉儒雅"要求

**解决方案**：
- 重写为 SVG 矢量绘图古风人物
- 包含发髻、木簪、交领长袍、甘草玉佩等细节

---

### 2.3 环境配置问题

| 问题 | 解决方式 |
|------|----------|
| 前端代理无法连接后端 | 确认后端 `pnpm dev` 已启动 |
| Vite 跨平台兼容 | 使用 `;` 分隔 Windows 命令 |
| 环境变量未生效 | 重启开发服务器 |
